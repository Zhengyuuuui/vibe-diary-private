用户个性化信息独立化功能开发文档
一、功能概述
1.1 功能描述
当前系统中所有用户的笔名和个性签名都使用了相同的默认值，导致用户个性化信息缺失。本功能旨在让每个用户拥有独立的笔名和个性签名，并设置合理的默认值规则。

1.2 问题分析
当前问题：数据库中 users 表的 pen_name 和 bio 字段设置了固定的默认值
pen_name 默认为 '林间漫步者'
bio 默认为 '在这数字时代的荒野，记录一抹温柔的呼吸。'
影响范围：所有新注册用户都会使用相同的默认值，无法体现个性化
1.3 需求目标
每个用户的 pen_name 和 bio 应该独立存储
pen_name 默认值应为用户的账号名（username）
bio 默认值应为 "这个家伙很神秘什么都没留下"
已存在的用户数据需要迁移更新
二、技术方案

function migrateUserProfiles(db) {  try {    const oldDefaultPenName = '林间漫步者';    const oldDefaultBio = '在这数字时代的荒    野，记录一抹温柔的呼吸。';        const users = db.prepare(      'SELECT id, username, pen_name, bio       FROM users WHERE pen_name = ? OR       bio = ?'    ).all(oldDefaultPenName,     oldDefaultBio);        if (users.length > 0) {      console.log(`发现 ${users.length} 个      用户需要迁移个性化数据...`);            const updateStmt = db.prepare(        'UPDATE users SET pen_name = ?,         bio = ? WHERE id = ?'      );            users.forEach(user => {        const newPenName = user.pen_name         === oldDefaultPenName ? user.        username : user.pen_name;        const newBio = user.bio ===         oldDefaultBio ? '' : user.bio;                updateStmt.run(newPenName,         newBio, user.id);        console.log(`已迁移用户 ${user.        username} 的数据`);      });            console.log('用户数据迁移完成');    }  } catch (error) {    console.error('用户数据迁移失败:',     error.message);  }}

四、数据迁移方案
4.1 迁移脚本
在 server/database.js 中添加迁移函数，并在初始化时自动执行。

4.2 迁移逻辑
查询所有 pen_name 为旧默认值 '林间漫步者' 的用户
将这些用户的 pen_name 更新为对应的 username
查询所有 bio 为旧默认值的用户
将这些用户的 bio 更新为空字符串（前端会显示新默认值）
4.3 迁移时机
在服务器启动时自动执行
只执行一次，通过判断字段值来避免重复迁移
