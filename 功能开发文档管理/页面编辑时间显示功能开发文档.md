# 页面编辑时间显示功能开发文档

## 需求概述

在日记页面底部添加"上次编辑时间"显示，格式为 `yyyy/MM/dd HH:mm:ss`，颜色需适配三种主题（light、dark、sepia）。

## ⚠️ 重要限制

**数据库修改限制**：
- ✅ 只允许添加新字段，禁止删除现有字段
- ✅ 只允许添加新索引，禁止删除现有索引
- ✅ 只允许添加新表，禁止删除现有表
- ✅ 必须保持现有数据的完整性

## 当前数据结构

### Pages 表（现有）

```sql
CREATE TABLE IF NOT EXISTS pages (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  diary_id INTEGER NOT NULL,
  content TEXT DEFAULT '',
  page_num INTEGER NOT NULL,
  FOREIGN KEY (diary_id) REFERENCES diaries(id) ON DELETE CASCADE
);
```

**缺失**：没有 `updated_at` 字段记录最后编辑时间。

## 修改方案

### 一、后端修改

#### 1. 数据库迁移 - 添加 updated_at 字段

**文件**：`server/database.js`

**修改位置**：在 `initDb` 函数中添加迁移逻辑

```javascript
function migratePagesUpdatedAt(db) {
  try {
    const tableInfo = db.prepare('PRAGMA table_info(pages)').all();
    const columns = tableInfo.map(col => col.name);
    
    if (!columns.includes('updated_at')) {
      console.log('开始为 pages 表添加 updated_at 字段...');
      
      // 添加字段，默认值为当前时间
      db.exec("ALTER TABLE pages ADD COLUMN updated_at TEXT DEFAULT (datetime('now', 'localtime'))");
      
      // 更新现有记录的时间戳
      db.exec("UPDATE pages SET updated_at = datetime('now', 'localtime') WHERE updated_at IS NULL");
      
      console.log('pages 表 updated_at 字段添加完成');
    }
  } catch (error) {
    console.error('pages 表迁移失败:', error.message);
  }
}
```

**在 `initDb` 函数末尾调用**：

```javascript
function initDb(dbPath) {
  // ... 现有代码 ...
  
  migrateOldUsers(db);
  migrateUserProfiles(db);
  migrateDiariesToUsers(db);
  migratePagesUpdatedAt(db);  // 新增：迁移 pages 表

  return db;
}
```

#### 2. 修改 PUT /pages/:id 接口

**文件**：`server/routes/pages.js`

```javascript
router.put('/:id', authMiddleware, (req, res) => {
  const db = req.db;
  const { content } = req.body;
  const page = db.prepare('SELECT p.* FROM pages p JOIN diaries d ON p.diary_id = d.id WHERE p.id = ? AND d.user_id = ?').get(req.params.id, req.user.id);
  if (!page) {
    return res.status(404).json({ code: 404, msg: '页面不存在或无权访问', data: null });
  }

  const safeContent = typeof content === 'string' ? sanitizeHtml(content) : page.content;
  
  // 修改：同时更新 content 和 updated_at
  db.prepare('UPDATE pages SET content = ?, updated_at = datetime(\'now\', \'localtime\') WHERE id = ?').run(safeContent, req.params.id);
  
  res.json({ code: 200, msg: '保存成功', data: null });
});
```

#### 3. 修改 POST /pages/:diaryId 接口（新增页面时设置时间）

**文件**：`server/routes/pages.js`

```javascript
router.post('/:diaryId', authMiddleware, (req, res) => {
  const db = req.db;
  const { content } = req.body;
  const diary = db.prepare('SELECT * FROM diaries WHERE id = ? AND user_id = ?').get(req.params.diaryId, req.user.id);
  if (!diary) {
    return res.status(404).json({ code: 404, msg: '日记本不存在或无权访问', data: null });
  }

  const maxPage = db.prepare('SELECT MAX(page_num) AS max FROM pages WHERE diary_id = ?').get(req.params.diaryId);
  const nextPage = (maxPage.max || 0) + 1;
  const safeContent = typeof content === 'string' ? sanitizeHtml(content) : '';

  // 修改：插入时包含 updated_at
  const result = db.prepare('INSERT INTO pages (diary_id, content, page_num, updated_at) VALUES (?, ?, ?, datetime(\'now\', \'localtime\'))').run(
    req.params.diaryId, safeContent, nextPage
  );
  res.status(201).json({ code: 201, msg: '页面创建成功', data: { id: result.lastInsertRowid } });
});
```

### 二、前端修改

#### 1. 添加时间格式化函数

**文件**：`client/src/views/DiaryReaderView.vue`

**位置**：`<script setup>` 中

```javascript
function formatDateTime(dateStr) {
  if (!dateStr) return ''
  const date = new Date(dateStr)
  if (isNaN(date.getTime())) return ''
  
  const year = date.getFullYear()
  const month = String(date.getMonth() + 1).padStart(2, '0')
  const day = String(date.getDate()).padStart(2, '0')
  const hours = String(date.getHours()).padStart(2, '0')
  const minutes = String(date.getMinutes()).padStart(2, '0')
  const seconds = String(date.getSeconds()).padStart(2, '0')
  
  return `${year}/${month}/${day} ${hours}:${minutes}:${seconds}`
}
```

## 数据库迁移安全说明

### 迁移脚本特性

| 特性 | 说明 |
|------|------|
| 幂等性 | 多次执行不会产生副作用 |
| 无损性 | 不删除任何现有数据 |
| 向后兼容 | 新字段有默认值，旧代码仍可运行 |
| 安全检查 | 先检查字段是否存在再添加 |

### 迁移 SQL 详解

```sql
-- 添加字段（如果不存在）
ALTER TABLE pages ADD COLUMN updated_at TEXT DEFAULT (datetime('now', 'localtime'));

-- 更新现有记录（可选，确保所有记录都有时间戳）
UPDATE pages SET updated_at = datetime('now', 'localtime') WHERE updated_at IS NULL;
```

**说明**：
- `ALTER TABLE ... ADD COLUMN` 只添加新列，不影响现有数据
- `DEFAULT` 约束确保新记录自动获得时间戳
- 现有记录的 `updated_at` 会被设置为迁移时的当前时间

## 注意事项

1. **数据库备份**：执行迁移前建议备份数据库文件
2. **时区处理**：使用 `datetime('now', 'localtime')` 获取本地时间
3. **空值处理**：前端需处理 `updated_at` 为空的情况
4. **性能影响**：时间戳更新是轻量级操作，对性能影响可忽略
