用户独立书架功能开发文档
一、功能概述
1.1 功能描述
当前系统中所有用户看到的日记本都是共享的，没有按用户隔离。本功能旨在为每个用户创建独立的书架，确保每个用户只能看到和管理自己的日记本。

1.2 问题分析
当前问题：

diaries 表缺少 user_id 字段，无法关联用户
日记相关的 API 接口没有使用认证中间件
所有用户都能看到所有的日记本，缺乏数据隔离
影响范围：

书架页面显示所有日记本
日记本的创建、编辑、删除没有用户权限控制
数据安全性问题：用户可以操作其他用户的日记
1.3 需求目标
每个日记本关联到创建它的用户
用户只能查看、编辑、删除自己的日记本
已存在的日记本需要迁移到特定用户或清理
确保数据隔离和安全性
二、技术方案
2.1 数据库层面修改
2.1.1 修改 diaries 表结构
文件位置：server/database.js

修改内容：

2.1.2 数据迁移策略
需要处理已存在的日记本数据：

方案一：将所有现有日记分配给默认用户（如第一个用户）
方案二：删除所有现有日记（清空数据重新开始）
推荐方案：方案一，保留数据并分配给默认用户
