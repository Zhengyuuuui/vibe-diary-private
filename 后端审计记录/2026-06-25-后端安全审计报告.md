
3. 生产环境使用强随机密钥（至少 32 字节）

**✅ 实际修复代码已应用（2026-06-25）**:

### 问题 2: 缺少事务保护导致数据一致性风险 ✅ 已修复（后端 Agent - 2026-06-25）

**位置**: `server/routes/diaries.js` 第 197-201 行

```javascript
const result = db.prepare(
  'INSERT INTO diaries (user_id, title, cover_style) VALUES (?, ?, ?)'
).run(req.user.id, sanitizeHtml(title), style);

db.prepare('INSERT INTO pages (diary_id, content, page_num) VALUES (?, ?, ?)').run(result.lastInsertRowid, '', 1);
```

**问题说明**:
- 创建日记和创建第一页是两个独立操作，无事务保护
- 如果第二个 INSERT 失败，会出现"空日记本"（无页面）
- 数据库状态不一致

**影响范围**: 
- 新创建的日记可能无法正常使用
- 用户数据完整性受损

**修复建议**:
使用 better-sqlite3 的事务包装：

**✅ 实际修复代码已应用（2026-06-25）**:
- 使用 `db.transaction()` 包装创建日记和页面操作
- 添加 try-catch 异常处理
- 测试验证：创建日记时页面自动创建，事务保护生效

```javascript
// server/routes/diaries.js
router.post('/', authMiddleware, (req, res) => {
  const db = req.db;
  const { title, cover_style } = req.body;
  
  if (!title || title.length > 20) {
    return res.status(400).json({ code: 400, msg: '标题必填且不超过20字', data: null });
  }
  
  const validStyles = ['leather', 'linen', 'pattern'];
  const style = validStyles.includes(cover_style) ? cover_style : 'leather';

  try {
    const createDiary = db.transaction((userId, title, style) => {
      const result = db.prepare(
        'INSERT INTO diaries (user_id, title, cover_style) VALUES (?, ?, ?)'
      ).run(userId, title, style);
      
      db.prepare('INSERT INTO pages (diary_id, content, page_num) VALUES (?, ?, ?)').run(result.lastInsertRowid, '', 1);
      
      return result.lastInsertRowid;
    });

    const diaryId = createDiary(req.user.id, sanitizeHtml(title), style);
    res.status(201).json({ code: 201, msg: '创建成功', data: { id: diaryId } });
  } catch (error) {
    console.error('创建日记失败:', error);
    res.status(500).json({ code: 500, msg: '创建失败', data: null });
  }
});
```

### 问题 3: 删除操作缺少事务保护 ✅ 已修复（后端 Agent - 2026-06-25）

**位置**: `server/routes/diaries.js` 第 228-229 行

```javascript
db.prepare('DELETE FROM pages WHERE diary_id = ?').run(req.params.id);
db.prepare('DELETE FROM diaries WHERE id = ?').run(req.params.id);
```

**问题说明**:
- 手动删除 pages 后删除 diaries，但数据库已配置 `ON DELETE CASCADE`
- 存在冗余操作，且两个 DELETE 之间无事务
- 如果第一个 DELETE 成功但第二个失败，pages 数据丢失但 diary 记录残留

**影响范围**: 
- 数据不一致
- orphaned pages 记录

**修复建议**:
1. 依赖数据库的 CASCADE 自动删除 pages（推荐）
2. 或使用事务包装：

**✅ 实际修复代码已应用（2026-06-25）**:
- 移除手动删除 pages 的冗余代码
- 使用 `db.transaction()` 包装删除操作
- 依赖数据库 CASCADE 自动删除 pages
- 添加 try-catch 异常处理
- 测试验证：删除日记时页面通过 CASCADE 自动删除

```javascript
// server/routes/diaries.js
router.delete('/:id', authMiddleware, (req, res) => {
  const db = req.db;
  const diary = db.prepare('SELECT * FROM diaries WHERE id = ? AND user_id = ?').get(req.params.id, req.user.id);
  
  if (!diary) {
    return res.status(404).json({ code: 404, msg: '日记本不存在或无权访问', data: null });
  }

  try {
    const deleteDiary = db.transaction((diaryId) => {
      // 只删除 diary，pages 会通过 CASCADE 自动删除
      db.prepare('DELETE FROM diaries WHERE id = ?').run(diaryId);
    });
    
    deleteDiary(req.params.id);
    res.json({ code: 200, msg: '删除成功', data: null });
  } catch (error) {
    console.error('删除日记失败:', error);
    res.status(500).json({ code: 500, msg: '删除失败', data: null });
  }
});
```

- 帮助攻击者了解系统结构

- 生产环境：console.error('Error:', err.message)，不泄露堆栈
- 开发环境：console.error(err.stack)，保留完整堆栈

```javascript
  }
  res.status(500).json({ code: 500, msg: '服务器内部错误', data: null });
```

### 问题 7: CORS 配置过于宽松 ✅ 已修复（后端 Agent - 2026-06-25）

**位置**: `server/index.js` 第 14 行

```javascript
app.use(cors());  // 无限制，允许所有来源
```

**问题说明**:
- 默认 CORS 配置允许所有来源访问
- 生产环境应限制可信域名

**影响范围**: 
- CSRF 风险增加
- 跨域攻击可能性

**修复建议**:
```javascript
// server/index.js
const allowedOrigins = process.env.ALLOWED_ORIGINS?.split(',') || ['http://localhost:5173'];

app.use(cors({
  origin: (origin, callback) => {
    if (!origin || allowedOrigins.includes(origin)) {
      callback(null, true);
    } else {
      callback(new Error('不允许的来源'));
    }
  },
  credentials: true
}));
```

**✅ 实际修复代码已应用（2026-06-25）**:
- 限制 CORS 只允许可信域名
- 支持通过环境变量 ALLOWED_ORIGINS 配置可信域名列表
- 默认允许 http://localhost:5173 和 http://localhost:3000（开发环境）
- 测试验证：允许域名访问成功，不允许域名被拒绝
