## 二、核心业务流程

> **⚠️ 唯一主流程，严禁新增扩展，Vibe Coding 仅围绕此流程开发**

### 主流程
进入虚拟书架(P-001) → 点击日记本封面 → 模拟书本翻开动画(P-002) → 沉浸式书写与物理翻页(F-002) → 自动保存并退出


### 关键异常流程
| 编号 | 异常场景 | 预期处理 |
| ---- | -------- | -------- |
| EX-001 | 本地 StPageFlip 库加载冲突 | 降级为平滑滑动切换，并提示“正在寻找纸张感...” |
| EX-002 | 数据库连接失败（SQLite 锁定） | 提示“日记本被占用”，尝试 3 次重连，失败后引导用户检查文件权限 |
| EX-003 | 登录态过期（未来扩展） | 自动跳转登录页，提示"登录已过期，请重新登录" |

## 三、功能模块清单

### （一）前台核心功能

#### F-001：拟物化虚拟书架
- **核心逻辑**：严格复刻 Stitch 设计稿的视觉，展示日记本列表并支持响应式排列。
- **功能细节**：从 T-001 读取日记列表 → 渲染带堆叠阴影的封面 → 点击触发翻开动画。
- **边界规则**：单屏书本数量超过 20 本时采用懒加载；书名超长需进行省略处理。
- **关联数据表**：T-001

#### F-002：StPageFlip 沉浸式翻页引擎
- **核心逻辑**：调用本地 `StPageFlip-master` 库，将日记页面转化为具有物理质感的翻页组件。
- **功能细节**：初始化 Canvas/DOM 渲染 → 模拟纸张弯曲与背光阴影 → 支持拖拽边缘翻页。
- **边界规则**：禁止在翻页动画未结束时连续触发新动画；适配移动端触摸手势。
- **关联数据表**：T-002

#### F-003：日记内容编辑 (CRUD)
- **核心逻辑**：在翻页容器的每一页提供文本编辑区域。
- **功能细节**：输入内容 → 停止操作 2 秒后自动调用 API → 更新 T-002。
- **边界规则**：单页字数限制 2000 字以防性能卡顿。
- **关联数据表**：T-002

#### F-004：封面材质与样式自定义
- **核心逻辑**：允许用户修改日记本的视觉属性。
- **功能细节**：打开设置面板 → 选择材质（皮革/亚麻） → 更新 T-001 样式字段 → 实时预览。
- **关联数据表**：T-001

## 五、数据字段规范

### T-001：diaries（日记本主表）

| 字段名 | 类型 | 必填 | 说明 | 约束 |
| ------ | ---- | ---- | ---- | ---- |
| id | 整数 | ✅ | 唯一主键 | 自增 |
| title | 字符串 | ✅ | 日记本标题 | 长度 ≤ 20 |
| cover_style | 字符串 | ✅ | 封面样式 ID | 对应 CSS Class |
| created_at | 时间戳 | ✅ | 创建日期 | 默认当前时间 |

**表关系说明**：
- T-001 与 T-002 的关系：一对多，通过 `diary_id` 关联。

### T-002：pages（日记页面表）

| 字段名 | 类型 | 必填 | 说明 | 约束 |
| ------ | ---- | ---- | ---- | ---- |
| id | 整数 | ✅ | 唯一主键 | 自增 |
| diary_id | 整数 | ✅ | 所属日记本 ID | 外键关联 T-001 |
| content | 文本 | ❌ | 页面内容 | |
| page_num | 整数 | ✅ | 物理页码序号 | 用于 StPageFlip 排序 |
