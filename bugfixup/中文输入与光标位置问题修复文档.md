# Contenteditable 中文输入与光标位置问题修复文档

## 问题描述

在编辑模式下使用 `contenteditable` 时出现两个问题：
1. **无法输入汉字**：使用中文输入法时，拼音无法正常转换为汉字
2. **光标位置错误**：每次输入后，光标跳转到文本开头

## 问题根源分析

### 问题1：中文输入法冲突

**原因**：
- 中文输入法在输入过程中会触发 `compositionstart` → 多次 `input` → `compositionend` 事件序列
- 当前代码在每次 `input` 事件时都执行 `handleInput`
- 在输入法组合过程中修改 DOM 或数据，会中断输入法的正常工作

**事件序列**：
用户输入 "你好"：

1. compositionstart (开始组合)
2. input (输入 "n")      ← handleInput 被调用，干扰输入法
3. input (输入 "ni")     ← handleInput 被调用，干扰输入法
4. input (输入 "nin")    ← handleInput 被调用，干扰输入法
5. input (输入 "你")     ← handleInput 被调用，干扰输入法
6. ...
7. compositionend (组合结束)
### 问题2：Vue 响应式更新导致 DOM 重置

**代码位置**：模板第 494 行

```vue
<div contenteditable="true" ...>
  {{ pageContents[page.id] ?? page.content }}
</div>
```

**原因**：
- Vue 的插值表达式 `{{ }}` 是响应式绑定
- 每次 `pageContents` 更新时，Vue 会重新计算并更新 DOM 内容
- 这会导致 `contenteditable` 元素的整个 innerHTML 被替换
- 光标位置和选区状态因此丢失，重置到开头

**数据流**：
用户输入 → input 事件 → handleInput()
→ pageContents.value[pageId] = content (Vue 响应式更新)
→ DOM 内容被替换
→ 光标位置丢失 → 光标跳到开头
## 修复方案

### 核心思路

1. **使用 `isComposing` 标志**：跟踪输入法组合状态，组合过程中不处理 input
2. **移除响应式内容绑定**：contenteditable 只设置初始值，之后由 DOM 自己管理内容
3. **使用 `v-once` 或手动控制**：避免 Vue 重新渲染 contenteditable 内容

### 修复代码

#### 1. 添加输入法状态跟踪

```javascript
// 在 script setup 顶部添加
const isComposing = ref(false)
```

#### 2. 添加组合事件处理函数

```javascript
function handleCompositionStart() {
  isComposing.value = true
}

function handleCompositionEnd(event) {
  isComposing.value = false
  // 组合结束后，手动触发一次内容更新
  const pageId = editingPageId.value
  if (pageId) {
    const content = event.target.innerText || ''
    pageContents.value[pageId] = content
  }
}
```

#### 3. 修改 handleInput 函数

```javascript
function handleInput(pageId, event) {
  // 如果正在使用输入法组合，不处理
  if (isComposing.value) {
    return
  }
  
  const content = event.target.innerText || ''
  
  if (content.length > charLimit) {
    event.target.innerText = content.substring(0, charLimit)
    return
  }
  
  pageContents.value[pageId] = content
  
  saveStatus.value = 'saving'
  showSaveToast.value = true
  
  if (saveTimers.value[pageId]) {
    clearTimeout(saveTimers.value[pageId])
  }
  
  saveTimers.value[pageId] = setTimeout(async () => {
    try {
      await savePage(pageId, content)
      saveStatus.value = 'saved'
      
      const pageIndex = store.currentDiary.pages.findIndex(p => p.id === pageId)
      if (pageIndex !== -1) {
        store.currentDiary.pages[pageIndex].content = content
      }
      
      delete saveTimers.value[pageId]
      
      setTimeout(() => {
        showSaveToast.value = false
        saveStatus.value = ''
      }, 3000)
    } catch (err) {
      console.error('自动保存失败:', err)
      saveStatus.value = ''
    }
  }, 2000)
}
```

#### 4. 修改模板（移除响应式绑定）

```vue
<div
  v-if="isEditMode"
  :data-page-id="page.id"
  contenteditable="true"
  class="flex-1 font-body text-on-surface text-sm leading-relaxed outline-none whitespace-pre-wrap break-words overflow-y-auto"
  :class="{ 'pointer-events-none': isFlipping }"
  @input="handleInput(page.id, $event)"
  @compositionstart="handleCompositionStart"
  @compositionend="handleCompositionEnd"
  @focus="editingPageId = page.id"
  @click="editingPageId = page.id"
></div>
```

**关键变化**：
- ❌ 移除 `>{{ pageContents[page.id] ?? page.content }}</div>`
- ✅ 使用自闭合标签 `></div>` 或空内容
- ✅ 添加 `@compositionstart` 和 `@compositionend` 事件

#### 5. 添加初始化内容的逻辑

当页面进入编辑模式时，需要手动设置 contenteditable 的初始内容：

```javascript
function handlePageDoubleClick(pageId) {
  if (!isEditMode.value) {
    toggleToEditMode()
    editingPageId.value = pageId
    
    nextTick(() => {
      const editArea = document.querySelector(`[data-page-id="${pageId}"]`)
      if (editArea) {
        // 设置初始内容，但不触发响应式更新
        editArea.innerText = pageContents.value[pageId] ?? store.currentDiary.pages.find(p => p.id === pageId)?.content ?? ''
        editArea.focus()
      }
    })
  }
}
```

#### 6. 修改 toggleToEditMode（可选优化）

```javascript
function toggleToEditMode() {
  isEditMode.value = true
  disablePageFlip()
  
  // 初始化当前可见页面的编辑内容
  nextTick(() => {
    const currentPage = store.currentDiary?.pages[currentPageIndex.value]
    if (currentPage && !pageContents.value[currentPage.id]) {
      pageContents.value[currentPage.id] = currentPage.content
    }
  })
}
```

## 技术要点总结

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 无法输入中文 | input 事件在输入法组合中被触发 | 使用 `isComposing` 标志跳过组合中的 input |
| 光标跳到开头 | Vue 响应式更新重置 DOM | 移除插值绑定，手动设置初始内容 |

## 注意事项

1. **不要在 contenteditable 上使用 `v-model`**：Vue 的 v-model 会同步更新 DOM，导致光标问题
2. **输入法事件顺序**：`compositionstart` → `input`×N → `compositionend`
3. **保存时机**：可以在 `compositionend` 后触发保存，确保中文输入完整
