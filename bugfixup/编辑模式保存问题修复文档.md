# 编辑模式保存问题修复文档

## 问题描述

用户在编辑模式下输入内容后：
1. 自动保存提示"已保存"
2. 点击"浏览"按钮时仍提示"有未保存的内容，是否保存？"
3. 点击确定后，页面显示为"空白页"
4. 刷新页面后，之前输入的内容又重新出现

## 问题根源分析

### 问题1：编辑模式下显示数据源不一致

**代码位置**：`DiaryReaderView.vue` 第 458-464 行

```vue
<div
  v-if="isEditMode"
  contenteditable="true"
  ...
>{{ page.content }}</div>
```

**问题**：
- 编辑模式的 `contenteditable` 初始值使用的是 `page.content`（来自 store）
- 但用户输入时，内容被存储在 `pageContents.value[page.id]` 中
- 这导致编辑区域显示的是 store 中的旧数据，而不是用户刚输入的新数据

### 问题2：保存成功后未更新 store

**代码位置**：`DiaryReaderView.vue` 第 156-169 行

```javascript
saveTimers.value[pageId] = setTimeout(async () => {
  try {
    await savePage(pageId, content)
    saveStatus.value = 'saved'
    // ❌ 缺少：更新 store 中的 page.content
    // ❌ 缺少：删除 saveTimers.value[pageId]
    
    setTimeout(() => {
      showSaveToast.value = false
      saveStatus.value = ''
    }, 3000)
  } catch (err) {
    console.error('自动保存失败:', err)
    saveStatus.value = ''
  }
}, 2000)
```

**问题**：
- 保存到后端成功了（刷新后数据存在）
- 但 store 中的 `page.content` 没有被更新
- 导致切换到浏览模式时显示旧数据

### 问题3：`hasUnsavedChanges()` 判断逻辑错误

**代码位置**：`DiaryReaderView.vue` 第 231 行

```javascript
function hasUnsavedChanges() {
  return Object.keys(saveTimers.value).length > 0
}
```

**问题**：
- 这个函数检查的是定时器对象是否有 key
- 但定时器执行完毕后，key 仍然存在于对象中（只是定时器已触发）
- 所以即使保存成功，这个函数仍返回 `true`

### 问题4：浏览模式显示数据源

**代码位置**：`DiaryReaderView.vue` 第 467-470 行

```vue
<div v-else ...>{{ page.content || '（空白页）' }}</div>
```

**问题**：
- 浏览模式显示的是 `page.content`（来自 store）
- 而 store 中的数据没有被更新
- 所以显示的是旧数据（空白）

## 数据流分析
用户输入 → pageContents.value[pageId] → savePage() → 后端数据库 ✅
↓
store.page.content ❌ (未更新)
↓
浏览模式显示 ❌ (显示旧数据)
## 修复方案

### 修复1：编辑模式显示正确的数据源

```vue
<!-- 修改前 -->
<div v-if="isEditMode" ...>{{ page.content }}</div>

<!-- 修改后 -->
<div v-if="isEditMode" ...>{{ pageContents[page.id] ?? page.content }}</div>
```

### 修复2：保存成功后更新 store

```javascript
saveTimers.value[pageId] = setTimeout(async () => {
  try {
    await savePage(pageId, content)
    saveStatus.value = 'saved'
    
    // ✅ 新增：更新 store 中的页面内容
    const pageIndex = store.currentDiary.pages.findIndex(p => p.id === pageId)
    if (pageIndex !== -1) {
      store.currentDiary.pages[pageIndex].content = content
    }
    
    // ✅ 新增：清除定时器记录
    delete saveTimers.value[pageId]
    
    setTimeout(() => {
      showSaveToast.value = false
      saveStatus.value = ''
    }, 3000)
  } catch (err) {
    console.error('自动保存失败:', err)
    saveStatus.value = ''
  }
}, 2000)
```

### 修复3：修正 `hasUnsavedChanges()` 逻辑

```javascript
function hasUnsavedChanges() {
  // 检查是否有正在等待保存的定时器
  // Object.values 返回的是定时器 ID，如果存在则表示有待保存的内容
  const pendingTimers = Object.values(saveTimers.value).filter(timer => timer !== null && timer !== undefined)
  return pendingTimers.length > 0
}
```

或者更简洁的方式：直接比较内容是否变化：

```javascript
function hasUnsavedChanges() {
  if (!store.currentDiary) return false
  
  for (const page of store.currentDiary.pages) {
    const localContent = pageContents.value[page.id] ?? ''
    const storeContent = page.content ?? ''
    if (localContent !== storeContent) {
      return true
    }
  }
  return false
}
```

### 修复4：`saveAllChanges` 也需要更新 store

```javascript
async function saveAllChanges() {
  for (const [pageId, content] of Object.entries(pageContents.value)) {
    try {
      await savePage(pageId, content)
      delete saveTimers.value[pageId]
      
      // ✅ 新增：更新 store
      const pageIndex = store.currentDiary.pages.findIndex(p => p.id === pageId)
      if (pageIndex !== -1) {
        store.currentDiary.pages[pageIndex].content = content
      }
    } catch (err) {
      console.error('保存失败:', err)
    }
  }
}
```

## 修复优先级

| 优先级 | 问题 | 影响 |
|--------|------|------|
| 🔴 高 | 保存后未更新 store | 导致浏览模式显示旧数据 |
| 🔴 高 | 编辑模式显示数据源错误 | 用户输入后看不到自己的内容 |
| 🟡 中 | `hasUnsavedChanges()` 逻辑错误 | 错误提示"有未保存内容" |
| 🟡 中 | `saveAllChanges()` 未更新 store | 手动保存后显示旧数据 |

## 总结

核心问题是**数据同步机制缺失**：
1. `pageContents` 是本地编辑状态
2. `store.currentDiary.pages` 是全局状态
3. 保存到后端成功后，需要同时更新全局状态
4. 否则切换模式时会显示不一致的数据
