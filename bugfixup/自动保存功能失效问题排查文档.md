# 自动保存功能失效问题排查文档

## 问题描述

自动保存功能有时候会失效，有时候页面只是自动保存一次就会失效。

## 代码分析

### 当前实现流程
用户输入 → input 事件 → handleInput()
↓
检查 isComposing（中文输入法状态）
↓
设置 2 秒延迟定时器
↓
执行保存

### 发现的问题

#### 问题 1：`isComposing` 状态可能卡住

**代码位置**：`handleInput` 函数

```javascript
function handleInput(pageId, event) {
  if (isComposing.value) {
    return  // 如果正在中文输入，直接返回
  }
  // ...
}
```

**问题**：
- 如果 `compositionend` 事件因某种原因没有触发
- `isComposing.value` 会一直为 `true`
- 后续所有 `handleInput` 都会直接返回，不执行保存

**可能触发场景**：
- 快速切换输入法
- 浏览器失去焦点时正在输入中文
- 某些特殊字符输入

#### 问题 2：`handleCompositionEnd` 没有触发保存

**代码位置**：`handleCompositionEnd` 函数

```javascript
function handleCompositionEnd(event) {
  isComposing.value = false
  const pageId = editingPageId.value
  if (pageId) {
    const content = event.target.innerText || ''
    pageContents.value[pageId] = content  // ❌ 只更新了数据，没有触发保存
  }
}
```

**问题**：
- 中文输入完成后，只更新了 `pageContents`
- 没有触发自动保存逻辑
- 用户输入中文后，内容不会被保存

#### 问题 3：`editingPageId` 类型不一致

**代码位置**：多处使用 `editingPageId`

```javascript
// handleCompositionEnd 中
const pageId = editingPageId.value  // 可能是数字

// handleInput 中
const pageIdStr = String(pageId)    // 转换为字符串

// pageContents 使用
pageContents.value[pageIdStr]       // 使用字符串作为 key
```

**问题**：
- `editingPageId` 的类型可能不一致
- 可能导致保存时找不到正确的页面

#### 问题 4：保存失败后定时器记录未清理

**代码位置**：`handleInput` 中的 catch 块

```javascript
saveTimers.value[pageIdStr] = setTimeout(async () => {
  try {
    await savePage(pageIdStr, content)
    // ...
    delete saveTimers.value[pageIdStr]  // 成功时删除
  } catch (err) {
    console.error('自动保存失败:', err)
    saveStatus.value = ''
    // ❌ 失败时没有删除定时器记录
  }
}, 2000)
```

**问题**：
- 保存失败后，`saveTimers.value[pageIdStr]` 仍然存在
- 但定时器已经执行完毕，值变成了一个无效的 timeout ID
- 后续的 `clearTimeout(saveTimers.value[pageIdStr])` 可能无法正常工作

## 修复方案

### 修复 1：添加 `isComposing` 安全重置

```javascript
function handleInput(pageId, event) {
  // 安全检查：如果 isComposing 为 true 超过 5 秒，强制重置
  // 这可以防止状态卡住
  
  if (isComposing.value) {
    return
  }
  // ...
}
```

### 修复 2：`handleCompositionEnd` 触发保存

```javascript
function handleCompositionEnd(event) {
  isComposing.value = false
  const pageId = editingPageId.value
  if (pageId) {
    const content = event.target.innerText || ''
    const pageIdStr = String(pageId)
    pageContents.value[pageIdStr] = content
    
    // ✅ 新增：触发保存逻辑
    triggerAutoSave(pageIdStr, content)
  }
}

// 抽取保存逻辑为独立函数
function triggerAutoSave(pageIdStr, content) {
  saveStatus.value = 'saving'
  showSaveToast.value = true
  
  if (saveTimers.value[pageIdStr]) {
    clearTimeout(saveTimers.value[pageIdStr])
  }
  
  saveTimers.value[pageIdStr] = setTimeout(async () => {
    try {
      await savePage(pageIdStr, content)
      saveStatus.value = 'saved'
      
      const pageIndex = store.currentDiary.pages.findIndex(p => String(p.id) === pageIdStr)
      if (pageIndex !== -1) {
        store.currentDiary.pages[pageIndex].content = content
        store.currentDiary.pages[pageIndex].updated_at = new Date().toISOString()
      }
      
      delete saveTimers.value[pageIdStr]
      
      setTimeout(() => {
        showSaveToast.value = false
        saveStatus.value = ''
      }, 3000)
    } catch (err) {
      console.error('自动保存失败:', err)
      saveStatus.value = ''
      delete saveTimers.value[pageIdStr]  // ✅ 失败时也删除
    }
  }, 2000)
}
```

### 修复 3：统一 `pageId` 类型处理

```javascript
// 在所有地方统一使用 String(pageId)
const pageIdStr = String(pageId)
```

### 修复 4：保存失败后清理定时器记录

```javascript
catch (err) {
  console.error('自动保存失败:', err)
  saveStatus.value = ''
```

## 完整修复代码

### 重构后的代码

```javascript
const isComposing = ref(false)
const composingStartTime = ref(null)

function handleCompositionStart() {
  isComposing.value = true
  composingStartTime.value = Date.now()
}

function handleCompositionEnd(event) {
  isComposing.value = false
  composingStartTime.value = null
  
  const pageId = editingPageId.value
  if (pageId) {
    const content = event.target.innerText || ''
    const pageIdStr = String(pageId)
    pageContents.value[pageIdStr] = content
    
    // 触发保存
    triggerAutoSave(pageIdStr, content)
  }
}

function handleInput(pageId, event) {
  // 安全检查：如果 isComposing 卡住超过 5 秒，强制重置
  if (isComposing.value && composingStartTime.value) {
    if (Date.now() - composingStartTime.value > 5000) {
      console.warn('⚠️ isComposing 状态卡住，强制重置')
      isComposing.value = false
      composingStartTime.value = null
    }
  }
  
  if (isComposing.value) {
    return
  }
  
  const content = event.target.innerText || ''
  
  if (content.length > charLimit) {
    event.target.innerText = content.substring(0, charLimit)
    return
  }
  
  const pageIdStr = String(pageId)
  pageContents.value[pageIdStr] = content
  
  triggerAutoSave(pageIdStr, content)
}

function triggerAutoSave(pageIdStr, content) {
  saveStatus.value = 'saving'
  showSaveToast.value = true
  
  if (saveTimers.value[pageIdStr]) {
    clearTimeout(saveTimers.value[pageIdStr])
  }
  
  saveTimers.value[pageIdStr] = setTimeout(async () => {
    try {
      await savePage(pageIdStr, content)
      saveStatus.value = 'saved'
      
      const pageIndex = store.currentDiary.pages.findIndex(p => String(p.id) === pageIdStr)
      if (pageIndex !== -1) {
        store.currentDiary.pages[pageIndex].content = content
        store.currentDiary.pages[pageIndex].updated_at = new Date().toISOString()
      }
      
      delete saveTimers.value[pageIdStr]
      
      setTimeout(() => {
        showSaveToast.value = false
        saveStatus.value = ''
      }, 3000)
    } catch (err) {
      console.error('自动保存失败:', err)
      saveStatus.value = ''
      delete saveTimers.value[pageIdStr]
    }
  }, 2000)
}
```

## 修复清单

| 问题 | 修复方案 |
|------|----------|
| `isComposing` 状态卡住 | 添加 5 秒超时强制重置 |
| 中文输入后不保存 | `handleCompositionEnd` 触发保存 |
| `pageId` 类型不一致 | 统一使用 `String(pageId)` |
| 保存失败后定时器记录未清理 | catch 块中添加 `delete` |
| 保存逻辑重复 | 抽取为 `triggerAutoSave` 函数 |
