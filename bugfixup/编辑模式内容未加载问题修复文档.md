# 编辑模式内容未加载问题修复文档

## 问题描述

点击"编辑"按钮进入编辑模式后：
- 编辑区域显示为空白
- 没有加载之前保存的内容
- 汉字输入和光标位置正常

## 问题根源分析

### 当前代码流程
点击"编辑"按钮 → toggleToEditMode()
↓
isEditMode.value = true
↓
watch(isEditMode) 触发
↓
nextTick() → setTimeout(50ms) → 更新 DOM

### 问题所在

**代码位置**：`toggleToEditMode()` 函数（第 252-261 行）

```javascript
function toggleToEditMode() {
  isEditMode.value = true
  disablePageFlip()
  
  nextTick(() => {
    const currentPage = store.currentDiary?.pages[currentPageIndex.value]
    if (currentPage && !pageContents.value[currentPage.id]) {
      pageContents.value[currentPage.id] = currentPage.content
    }
  })
}
```

**问题**：
1. ❌ 只更新了 `pageContents` 数据，没有更新 DOM
2. ❌ 条件判断 `!pageContents.value[currentPage.id]` 会阻止已有内容的更新
3. ❌ 没有设置 `editArea.innerText`

**代码位置**：`watch(isEditMode)` （第 332-352 行）

```javascript
watch(isEditMode, (newMode) => {
  if (newMode) {
    disablePageFlip()
    
    nextTick(() => {
      setTimeout(() => {
        const editAreas = document.querySelectorAll('[data-page-id]')
        editAreas.forEach(editArea => {
          // ... 更新 innerText
        })
      }, 50)
    })
  }
})
```

**问题**：
1. ❌ `setTimeout` 50ms 可能不够，DOM 可能还没渲染完成
2. ❌ `v-if="isEditMode"` 导致 DOM 元素在 `isEditMode = true` 后才创建
3. ❌ `nextTick` 只等待 Vue 响应式更新，但 `v-if` 的 DOM 创建可能需要更长时间

### 根本原因

模板中使用了 `v-if="isEditMode"`：

```vue
<div v-if="isEditMode" contenteditable="true" ...></div>
```

**时序问题**：
- isEditMode.value = true
- Vue 调度 DOM 更新（异步）
- watch 触发，执行 nextTick
- setTimeout 50ms 后尝试获取 DOM
- 但此时 v-if 创建的 DOM 可能还没完成渲染

## 修复方案

### 方案：在 toggleToEditMode 中直接更新 DOM

修改 `toggleToEditMode()` 函数，使用更可靠的 DOM 更新时机：

```javascript
function toggleToEditMode() {
  isEditMode.value = true
  disablePageFlip()
  
  // 使用 requestAnimationFrame 确保 DOM 已渲染
  requestAnimationFrame(() => {
    requestAnimationFrame(() => {
      const editAreas = document.querySelectorAll('[data-page-id]')
      editAreas.forEach(editArea => {
        const pageId = editArea.getAttribute('data-page-id')
        if (pageId) {
          const page = store.currentDiary?.pages.find(p => p.id === pageId)
          if (page) {
            const content = pageContents.value[pageId] ?? page.content ?? ''
            if (editArea.innerText !== content) {
              editArea.innerText = content
            }
          }
        }
      })
    })
  })
}
```

### 为什么使用双重 requestAnimationFrame

1. **第一个 requestAnimationFrame**：在下一帧渲染前执行
2. **第二个 requestAnimationFrame**：确保浏览器完成了 DOM 渲染

这比 `setTimeout` 更可靠，因为它与浏览器的渲染周期同步。

### 简化 watch 逻辑

由于 `toggleToEditMode()` 已经处理了 DOM 更新，可以简化 watch：

```javascript
watch(isEditMode, (newMode) => {
  if (newMode) {
    disablePageFlip()
    // DOM 更新已在 toggleToEditMode 中处理
  } else {
    enablePageFlip()
    editingPageId.value = null
  }
})
```

## 完整修复代码

### 修改 toggleToEditMode

```javascript

  isEditMode.value = true
  disablePageFlip()
  
  requestAnimationFrame(() => {

      editAreas.forEach(editArea => {
```

## 技术要点

| 方法 | 说明 | 可靠性 |
|------|------|--------|
| `nextTick` | Vue 响应式更新后执行 | ⚠️ 不保证 v-if DOM 已渲染 |
| `setTimeout(fn, 0)` | 宏任务，下一轮事件循环 | ⚠️ 可能不够 |
| `setTimeout(fn, 50)` | 延迟 50ms | ⚠️ 不可靠，取决于设备性能 |
| `requestAnimationFrame` | 下一帧渲染前 | ✅ 与渲染同步 |
| 双重 `requestAnimationFrame` | 确保渲染完成 | ✅✅ 最可靠 |

## 注意事项

1. **保持 handlePageDoubleClick 的逻辑**：双击进入编辑时也需要正确初始化内容
2. **不要恢复 Vue 插值绑定**：`{{ }}` 会导致光标问题
3. **compositionstart/end 事件保持不变**：确保中文输入正常
