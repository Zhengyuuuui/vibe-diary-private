# 页码显示优化开发文档

## 需求背景

当前页码显示存在问题：
- 封面被计入页码，显示为"第 1 页"
- 封底也被计入总页数
- 底部导航显示的页码包含了封面和封底

用户期望：
- 封面和封底不计入页码
- 内容页从第 1 页开始编号
- 底部导航只显示内容页的页码

## 当前实现分析

### 数据结构
displayPages 数组结构：
┌─────────────────────────────────────────────────────┐
│ index 0: 封面                                        │
│         - 来源：store.currentDiary.pages[0]          │
│         - 显示：日记标题、创建日期                    │
├─────────────────────────────────────────────────────┤
│ index 1 ~ n-1: 内容页                                │
│         - 来源：store.currentDiary.pages[1...n-1]    │
│         - 显示：page.content                         │
├─────────────────────────────────────────────────────┤
│ index n: 封底                                        │
│         - 动态添加，page.isBackCover = true          │
│         - 显示："氛围日记"                           │
└─────────────────────────────────────────────────────┘

### 当前页码计算

```javascript
// 当前实现
const totalPages = computed(() => {
  if (!pageFlipInstance.value) return 0
  return Math.max(0, pageFlipInstance.value.getPageCount() - 1)
})

// 底部导航显示
{{ currentPageIndex + 1 }} / {{ totalPages }}
```

**问题**：
- `totalPages` 减去了 1（封底），但没有减去封面
- `currentPageIndex + 1` 把封面算作第 1 页

## 修复方案

### 新增计算属性

```javascript
// 内容页总数（排除封面和封底）
const contentPageCount = computed(() => {
  if (!store.currentDiary?.pages) return 0
  // pages 包含封面，所以内容页数 = pages.length - 1
  return Math.max(0, store.currentDiary.pages.length - 1)
})

// 当前内容页码（1-based，0 表示封面或封底）
const currentContentPage = computed(() => {
  // currentPageIndex: 0=封面, 1=第1页, 2=第2页...
  if (currentPageIndex.value === 0) return 0  // 封面
  const lastPageIndex = pageFlipInstance.value?.getPageCount() - 1 || 0
  if (currentPageIndex.value >= lastPageIndex) return 0  // 封底
  return currentPageIndex.value  // 内容页码（1-based）
})

// 底部导航显示文本
const pageNavText = computed(() => {
  if (currentContentPage.value === 0) {
    return currentPageIndex.value === 0 ? '封面' : '封底'
  }
  return `${currentContentPage.value} / ${contentPageCount.value}`
})
```

## 注意事项

1. **index 与 page_num 的区别**：
   - `index` 是 displayPages 数组的索引（0=封面, 1=第1页...）
   - `page.page_num` 是数据库中存储的页码，可能与 index 不一致
   - 建议使用 `index` 作为显示依据

2. **翻页逻辑不受影响**：
   - `flipNext()` 和 `flipPrev()` 使用的是 `currentPageIndex`
   - 页码显示优化不影响翻页功能

3. **编辑模式兼容**：
   - 字数统计显示逻辑保持不变
   - 只修改页码显示部分
