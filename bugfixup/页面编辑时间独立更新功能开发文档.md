# 页面编辑时间独立更新功能开发文档

## 需求概述

实现每个页面的编辑时间独立显示：
- 修改第一页后，第一页显示修改后的时间
- 第二页未修改，保持原来的时间
- 每个页面的时间互不影响

## 问题分析

### 当前状态

| 层级 | 状态 | 说明 |
|------|------|------|
| 后端 | ✅ 正确 | PUT `/pages/:id` 会更新 `updated_at` |
| 数据库 | ✅ 正确 | 每个页面有独立的 `updated_at` 字段 |
| 前端 | ❌ 缺少同步 | 保存后未更新 store 中的 `updated_at` |

### 问题根源

```javascript
// 当前代码：保存成功后只更新了 content
if (pageIndex !== -1) {
  store.currentDiary.pages[pageIndex].content = content
  // ❌ 缺少：更新 updated_at
}
```

虽然后端已经更新了时间，但前端 store 中的 `updated_at` 没有同步更新，导致显示的还是旧时间。

## 解决方案

### 方案 A：手动更新前端时间（已选择）

保存成功后，手动更新 store 中的 `updated_at`。

**优点**：
- 实现简单，只需修改前端一行代码
- 性能好，无额外网络请求
- 时间差异可忽略（毫秒级）

## 修改方案

### 修改文件

**文件**：`client/src/views/DiaryReaderView.vue`

**修改位置**：`handleInput` 函数中的保存成功回调

### 修改前

```javascript
saveTimers.value[pageIdStr] = setTimeout(async () => {
  try {
    console.log('🚀 开始保存 - pageId:', pageIdStr, 'content:', content)
    await savePage(pageIdStr, content)
    saveStatus.value = 'saved'
    
    const pageIndex = store.currentDiary.pages.findIndex(p => String(p.id) === pageIdStr)
    
    if (pageIndex !== -1) {
      store.currentDiary.pages[pageIndex].content = content
      console.log('✅ 已更新 store.currentDiary.pages[' + pageIndex + '].content')
    } else {
      console.log('❌ 未找到对应页面')
    }
    
    delete saveTimers.value[pageIdStr]
    
    setTimeout(() => {
      showSaveToast.value = false
      saveStatus.value = ''
    }, 3000)
  } catch (err) {
    console.error('自动保存失败:', err)
    saveStatus.value = ''
  }
}, 2000)
```

### 修改后

```javascript
saveTimers.value[pageIdStr] = setTimeout(async () => {
  try {
    console.log('🚀 开始保存 - pageId:', pageIdStr, 'content:', content)
    await savePage(pageIdStr, content)
    saveStatus.value = 'saved'
    
    const pageIndex = store.currentDiary.pages.findIndex(p => String(p.id) === pageIdStr)
    
    if (pageIndex !== -1) {
      store.currentDiary.pages[pageIndex].content = content
      // ✅ 新增：同步更新 updated_at 时间
      store.currentDiary.pages[pageIndex].updated_at = new Date().toISOString()
      console.log('✅ 已更新 store.currentDiary.pages[' + pageIndex + '].content 和 updated_at')
    } else {
      console.log('❌ 未找到对应页面')
    }
    
    delete saveTimers.value[pageIdStr]
    
    setTimeout(() => {
      showSaveToast.value = false
      saveStatus.value = ''
    }, 3000)
  } catch (err) {
    console.error('自动保存失败:', err)
    saveStatus.value = ''
  }
}, 2000)
```

### 核心修改

```javascript
// 新增这一行
store.currentDiary.pages[pageIndex].updated_at = new Date().toISOString()
```

## 时间格式说明

### ISO 8601 格式

`new Date().toISOString()` 返回格式：`2026-04-13T15:30:00.000Z`

### formatDateTime 函数处理

现有的 `formatDateTime` 函数可以正确解析 ISO 格式：

```javascript
function formatDateTime(dateStr) {
  if (!dateStr) return ''
  const date = new Date(dateStr)  // 可以解析 ISO 格式
  // ...
  return `${year}/${month}/${day} ${hours}:${minutes}:${seconds}`
}
```

## 注意事项

1. **时区问题**：
   - `new Date().toISOString()` 返回 UTC 时间
   - 后端使用 `datetime('now', 'localtime')` 返回本地时间
   - `formatDateTime` 函数会自动转换为本地时间显示
   - 两者显示结果一致

2. **时间精度**：
   - 前端更新的时间与后端可能有毫秒级差异
   - 对用户体验无影响

3. **刷新页面**：
   - 刷新页面后，会从后端重新加载数据
   - 显示的是后端存储的准确时间
